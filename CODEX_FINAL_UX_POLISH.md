# 최종 UX 마감 및 레이아웃 버그 수정 지시서

Codex, 아래 3가지 항목은 사용자 경험에 직결되는 매우 중요한 디테일입니다. 이전 지시에서 누락되거나 잘못 구현된 부분이니 이번에는 1px의 오차 없이, 기술된 로직 그대로 정확히 구현하세요.

## 1. 버튼 피드백: '최소 로딩 시간' 보장 (Fake Delay UX)
- **대상:** 설정 탭의 `[앱 최신버전 업데이트]` 버튼과 `[지금 동기화]` 버튼.
- **문제:** 반영할 데이터가 없거나 처리가 너무 빨리 끝나면 사용자는 '버튼이 안 눌린 건가?' 하고 착각하게 됩니다.
- **해결 로직 (`app.js`):**
  - 버튼을 클릭하면 즉시 버튼을 비활성화(`disabled = true`)하고, 텍스트를 "동기화 중..." 또는 "업데이트 확인 중..."으로 변경하세요.
  - 실제 API 요청이나 워커 갱신 로직이 순식간에 끝나더라도, `await new Promise(resolve => setTimeout(resolve, 2000))` 로직을 강제로 삽입하여 **최소 2~3초간 진행 중인 상태를 유지**하세요.
  - 시간이 지나면 텍스트를 "완료!"로 1초간 띄웠다가 원래 텍스트로 복구하고 비활성화를 푸세요.

## 2. 업데이트 시간 표기: "마지막 확인: 방금 전"
- **대상:** 설정 탭의 `[앱 최신버전 업데이트]` 영역.
- **문제:** 동기화 시간은 표시되지만, 앱 업데이트 확인 시간은 표시되지 않아 답답합니다.
- **해결 로직:**
  - `동기화 상태` 영역처럼, 업데이트 버튼 위나 옆에 **"마지막 확인: 방금 전"** 이라는 텍스트 레이블(`<p id="update-time-label">`)을 추가하세요.
  - 업데이트 버튼을 눌렀을 때(또는 앱 최초 진입 시 서비스 워커 체크 완료 시)의 시간을 `localStorage.setItem('DONNAGA_LAST_UPDATE_CHECK', Date.now())`로 저장하고, 시간 포맷팅 함수(방금 전, 3분 전 등)를 거쳐 UI에 표시하세요.

## 3. 달력 아이콘 레이아웃 붕괴 완벽 해결 (CSS Position)
- **문제:** 수입, 저축, 투자 미니 아이콘이 추가될 때, 아이콘이 공간을 차지하면서 날짜 숫자가 밀리고 달력 열(Grid Column)의 정렬이 어긋나는 현상이 발생합니다.
- **해결 (절대 좌표 사용):**
  - 미니 아이콘이 기존 텍스트(날짜, 금액)를 절대 밀어내지 못하게 만들어야 합니다.
  - `styles.css`에서 달력 셀(`.calendar-day`)에 반드시 `position: relative;`를 부여하세요.
  - 주입되는 미니 아이콘(또는 아이콘들을 감싸는 컨테이너)에는 `position: absolute; top: 4px; right: 4px;` (또는 적절한 우측 상단 좌표)를 부여하여, 날짜 숫자 위로 **공중에 띄우세요**.
  - 이렇게 하면 아이콘이 1개든 3개든 달력의 높이나 너비 레이아웃이 절대 틀어지지 않습니다.
