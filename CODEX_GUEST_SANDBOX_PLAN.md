# 포트폴리오 공개용 샌드박스 및 이중 보안 아키텍처 지시서

가계부 앱을 외부(GitHub, LinkedIn)에 공개하기 위해, 관리자의 진짜 데이터를 완벽히 은닉하고 Guest에게는 체험용 더미 데이터를 제공하는 '이중 보안 및 샌드박스' 아키텍처를 구현합니다.

## 1. 보안 게이트(Login Gate) 이중 트랩 및 방어 로직 (`app.js`)
현재 4자리로 제한된 PIN 입력창을 교묘하게 설계하여 관리자만 진짜 암호를 입력할 수 있게 합니다.

- **이중 트랩 UI:** 입력창(input)의 `placeholder`나 안내 문구는 여전히 **"4자리 PIN 입력"**으로 유지하되, `maxlength` 속성을 **`8`**로 조용히 상향하세요. (GUEST는 0000을 입력하지만, 관리자는 8자리를 입력할 수 있습니다.)
- **브루트포스 방어 (Lockout):** 
  - `localStorage.getItem('DONNAGA_LOGIN_FAILS')`를 통해 실패 횟수를 추적하세요.
  - 3회 연속 틀리면 `localStorage.setItem('DONNAGA_LOCK_UNTIL', Date.now() + 60000)`을 설정하고, 입력창을 1분간 비활성화(`disabled`)한 뒤 "3회 오류로 1분간 잠깁니다"라는 카운트다운 UI를 보여주세요.

## 2. 권한 분리 및 데이터 소스(Source) 라우팅
PIN 번호에 따라 읽고 쓰는 데이터베이스(Source)를 완전히 분리합니다.

### A. Admin 모드 (PIN: `20200501`)
- **인증:** 입력값이 `20200501`이면 (서버 API `/api/verify-pin`를 거쳐 확인 후) `DONNAGA_ROLE='admin'`으로 로컬에 저장합니다.
- **동작:** 기존과 완벽하게 동일합니다. 모든 쓰기/조회 로직이 Cloudflare D1 서버(`/api/sync`)와 동기화됩니다.

### B. Guest 쇼케이스 모드 (PIN: `0000`)
- **인증:** 입력값이 `0000`이면 `DONNAGA_ROLE='guest'`로 로컬에 저장합니다.
- **설정 탭 원천 차단 (보안 강화):** Guest 모드로 진입 시 하단 탭바의 **`[설정]` 탭 버튼을 DOM에서 완전히 삭제하거나 `display: none !important;`로 숨김 처리**하세요. 화면 전환 이벤트 리스너에서도 Guest가 'settings' 화면에 강제 진입하려는 시도를 무시하도록 방어 로직을 넣어, 개발자 도구를 쓰더라도 절대 설정 화면을 볼 수 없게 막아야 합니다.
- **차단:** Guest 모드일 경우, `pushToServer`나 `pullFromServer` 같은 **모든 D1 동기화(Fetch API) 로직의 실행을 원천 차단(`return;`)**하세요. 서버와 절대 통신해서는 안 됩니다.
- **더미 데이터 주입 (Seed):** 
  - Guest 최초 접속 시, 로컬 IndexedDB가 비어있다면 `/guest/dummy-data.js` 파일에서 데이터를 가져와 IndexedDB에 삽입(Seed)하세요.
  - **가상화 처리:** 더미 데이터의 `date` 연/월(YYYY-MM) 부분을, 사용자가 앱을 켠 **현재 접속 시점의 연/월**로 치환하여 저장하세요. (예: 데이터가 2023-01 기반이더라도, 오늘 접속하면 2026-04 데이터로 변환되어 저장됨)
- **자율 체험:** Guest는 로컬 IndexedDB를 기반으로 내역 추가, 수정, 삭제 기능을 **100% 정상적으로 체험**할 수 있어야 합니다. (단지 새로고침 하거나 다른 브라우저로 켜면 본인의 로컬 데이터만 보일 뿐입니다.)

## 3. 백엔드 보안 수정 (`functions/api/verify-pin.js` 등)
- 클라우드플레어 대시보드의 `ADMIN_PIN` 환경 변수 값을 `20200501`로 변경할 예정입니다.
- 서버 API는 `20200501`이 아니면 D1 접근을 절대 허용하지 않도록 기존 보안 로직을 굳건히 유지하세요.

## 4. 더미 데이터 파일 생성 (`/guest/dummy-data.js`)
- 프로젝트 루트에 `/guest` 폴더를 만들고 `dummy-data.js` 파일을 생성하세요.
- 이 파일은 `export const dummyTransactions = [...]` 형태로 약 20~30건 정도의 그럴듯한 식비, 교통비, 카페, 수입 내역(현재 연/월 기준 텍스트 치환을 고려한 템플릿 형태)을 포함해야 합니다.
