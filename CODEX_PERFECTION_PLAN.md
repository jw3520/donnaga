# 앱 완성도 극대화 및 보안 시스템 구축 지시서

가계부의 디테일을 다듬고, 권한 없는 사용자의 데이터 훼손을 막기 위한 보안 시스템을 구축합니다. 아래 사항을 1px의 오차, 1줄의 버그 없이 완벽하게 구현하세요.

## 1. 설정 탭 UX Writing (사용자 친화적 용어)
설정(`settings`) 화면의 투박한 개발자 용어를 사용자 친화적으로 전면 수정하세요.
- **웹 캐시 삭제** → **`앱 최신버전 업데이트`** (설명: "앱 화면이 이상하거나 최신 반영이 안 될 때 누르세요.")
- **데이터 저장** → **`기록된 내역 수`** (값: IndexedDB의 전체 트랜잭션 개수 표기, 예: "총 342건")
- **공유 DB** → **`클라우드 연결 상태`**
- **최근 동기화** → **`마지막 동기화`** (값: "방금 전", "3분 전", "24.04.19 14:00" 등으로 친절하게 표기)
- **앱 설치** → **`홈 화면에 추가`**

## 2. 모바일 UI 줄바꿈(텍스트) 최적화
- **문제:** 갤럭시 S25 울트라 등 다양한 화면비에서 버튼 텍스트("지금 동\n기화")가 두 줄로 깨집니다.
- **해결 (CSS):** 모든 버튼(`.primary-button`, `.secondary-button`, `.ghost-button`, `.chip-button`, `.type-chip` 등)에 `white-space: nowrap;` 및 `word-break: keep-all;`을 추가하여 **절대 텍스트가 줄바꿈되지 않도록** 강제하세요.
- 여백이 부족하면 텍스트 크기를 살짝 줄이거나 버튼 좌우 패딩을 줄여 한 줄에 맞추세요.

## 3. 최초 1회 로그인 및 권한 분리 아키텍처 (Gatekeeper)
앱 진입 시 로컬 스토리지(`localStorage`)의 인증 상태를 확인하고, 권한(Role)에 따라 UI를 분기합니다.

### A. 프론트엔드 (초기 진입 및 권한 제어)
- **로그인 게이트:** 앱 실행 시 `localStorage.getItem('DONNAGA_PIN')`이 없으면 메인 화면을 가리는 **전체 화면 로그인 다이얼로그(Login Gate)**를 띄우세요. (취소 불가)
- **PIN 규칙 및 로컬 저장:**
  - 사용자가 **`0501`** 입력 시: `DONNAGA_PIN`에 `0501`을 저장하고 **관리자(Admin) 권한**을 부여합니다.
  - 사용자가 **`0000`** 입력 시: `DONNAGA_PIN`에 `0000`을 저장하고 **조회(Read-only) 권한**을 부여합니다.
  - 그 외 번호 입력 시: "비밀번호가 틀렸습니다." 에러 메시지를 표시합니다.
- **권한별 UI 렌더링:**
  - **Read-only (`0000`) 접속 시:** 하단 `(+) 내역 추가` FAB 버튼, 리스트의 `수정/삭제` 버튼, 설정 탭의 `강제 동기화` 등 쓰기 관련 UI를 `display: none`으로 완전히 숨기세요.
  - **Admin (`0501`) 접속 시:** 모든 기능이 정상적으로 활성화됩니다.
- **자동 로그인:** 한 번 PIN이 로컬에 저장되면 이후 앱을 껐다 켜도 비밀번호를 묻지 않고 해당 권한으로 즉시 진입합니다.

### B. 백엔드 (Cloudflare D1 동기화 - `/api/sync.js`)
- **쓰기(POST) 권한 통제:** 프론트엔드에서 데이터를 서버로 `POST`할 때, `localStorage`에 있는 PIN 값을 헤더나 Body(`pin: '0501'`)에 담아 보냅니다.
- **서버 검증:** 서버는 `context.env.SECRET_PIN` (값: `0501`)과 클라이언트가 보낸 `pin`을 비교합니다.
- **보안 차단:** 전송된 `pin`이 `0501`이 아니라면(예: `0000`이거나 위조된 경우) 서버는 즉시 `401 Unauthorized` 에러를 반환하고 D1 DB 쓰기/수정 작업을 차단해야 합니다. GET(조회) 요청은 핀 번호와 무관하게 허용합니다.

## 4. PWA 업데이트 알림 및 유도 UX (Service Worker Update)
앱의 새로운 버전이 배포되었을 때, 사용자가 즉시 인지하고 캐시를 갱신할 수 있도록 시각적인 유도 로직을 추가하세요.

- **업데이트 감지 로직:** `app.js`에서 서비스 워커 등록 시, 새로운 워커가 대기 중(`waiting`)이거나 업데이트(`updatefound`)가 감지되면 전역 상태를 변경하세요.
- **시각적 강조 (Highlighting):**
  1. 하단 탭바의 **[설정] 아이콘 우측 상단에 빨간색 점(Red Dot)**을 띄워 알림이 있음을 직관적으로 알리세요.
  2. 설정 탭 내부의 **`[앱 최신버전 업데이트]`** 버튼을 평소에는 `secondary-button`(눈에 띄지 않게)으로 두다가, 업데이트가 있을 때만 `primary-button`으로 색상을 바꾸고 크기를 살짝 키우는 등(Pulse 애니메이션 추천) 시선을 강하게 끌도록 만드세요.
- **업데이트 실행 로직:** 사용자가 버튼을 누르면 대기 중인 서비스 워커에 `skipWaiting` 메시지를 보내고, 즉시 `location.reload()`를 실행하여 최신 버전을 띄우세요.
- **원상 복구:** 업데이트가 완료되어 새로 켜지면 설정 탭의 빨간 점과 버튼 강조 표시가 모두 사라져야 합니다.

## 5. 개발자 주의사항
- Cloudflare 환경변수(`SECRET_PIN`)는 대시보드에 `0501`로 세팅할 것입니다.
- 설정 탭 가장 하단에 **[로그아웃 (권한 초기화)]** 버튼을 작게 만들어, 누르면 `localStorage`의 PIN을 지우고 앱을 새로고침하게 만드세요.
